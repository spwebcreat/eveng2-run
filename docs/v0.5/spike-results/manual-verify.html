3. なぜ v0.5.0 / v0.5.1 が動かなかったか

原因

spike worktree の vite.config.ts で 本番 entry と spike entry を両方ビルドする設定になっており、dist/ に本番の index.html + main-*.js (本番 UI bundle) が混入していました。app.json で entrypoint: spike-sN.html を指定しても、Hub OS は dist/index.html を優先起動する仕様のため、本番 UI が表示される結果に。

これは秘書 (Claude) が dispatch packet で「本番 index.html を温存」と発注した 秘書側の設計バグです。dev-engineer の実装過失ではありません。

修正内容

  1. spike worktree から本番 index.html を削除
  2. spike-sN.html を index.html にリネーム（spike が唯一の entry に）
  3. vite.config.ts を最小化（input 指定削除 → vite はデフォルトで index.html だけビルド）
  4. rm -rf dist → clean build → 検査ゲート 6 項目 PASS → pack
  5. version bump (S1: 0.5.0 → 0.5.2 / S4: 0.5.1 → 0.5.3) で Hub Portal cache 回避

結果: ehpk サイズが 46-47 KB → 33 KB に縮小（本番 bundle 除去の証拠）。

7. DevTools 接続手順（補助・任意）

📱 多くの場合 iPhone STATUS 表示だけで GO/NO-GO 判定可能。下記の DevTools 接続は補助手段で、必須ではありません（2026-05-28 旦那様実機検証で実証）。
⚠ Even Hub アプリの WKWebView 制限: 開発メニュー → iPhone を選んでも「調査可能なアプリケーションがありません」と出る場合があります。これは Even Hub アプリ側が iOS 16.4+ の WKPreferences.isInspectable = true を設定していないため（旦那様や秘書では解決不能）。その場合は次節「iPhone STATUS 表示で判定」を使ってください。

iPhone Safari Web Inspector で [s1] / [s4] ログを見る

  1. Mac 側準備: Safari → 設定 → 詳細 → ✅「メニューバーに開発メニューを表示」
  2. iPhone 側準備: 設定 → Safari → 詳細 → ✅「Web インスペクタ」
  3. iPhone を Mac に USB 接続（信頼ダイアログがあれば許可）
  4. Mac Safari の 開発メニュー → iPhone 名 → spike アプリ（G2 HUD Spike S1 または G2 Spike S4 Mic）を選択
  5. Web Inspector が起動 → コンソールタブで [s1] / [s4] プレフィックスのログを確認

📊 iPhone STATUS 表示で GO/NO-GO 判定（DevTools 不要）

iPhone Hub アプリ画面の STATUS 表示で必要数値はほぼ全て見えます。S4 を例に判定方法:

STATUS: MIC 35200 B/s 38ms ago | n=198 t=20s
              ↑avgBps     ↑直近    ↑events ↑経過秒
STATUS 項目計算 / GO 基準
NNN B/savgBps が 32,000 ± 10% (28.8K-35.2K) なら ✅
n=NNevents 数。5 分 (300s) で n≈3000 なら理論値通り ✅
t=NNs経過秒。done (timeout) で 5 分完走確認
NN ms ago直近 event からの経過。< 500ms なら stream 安定 ✅
派生: events/secn ÷ t ≈ 10/s が理論値 (100ms 間隔)
派生: bytes/event(B/s) ÷ (events/s) ≈ 3,200 が理論値
S1 の場合は STATUS でなく グラスの 2×2 タイル像の視覚チェック が判定ベースです（DevTools のログも、見えれば便利ですが必須ではない）。

8. トラブルシューティング

症状原因候補対処
グラスに本番 UI (READY / RUN) が見える (a) iPhone Hub アプリで本番アプリを起動した
(b) build 混入バグ再発		 iPhone Hub アプリのタイトル確認 → spike を選び直し / それでも本番なら秘書に報告
グラスに何も見えない（真っ黒） (a) bridge 取得 timeout
(b) attach 失敗
(c) ペアリング切れ		 DevTools コンソールで [s1] bridge timeout / [s1] attach failed 等のログを確認 → 秘書に共有
パターン切替が起きない（A だけ表示） (a) render エラー (B/C 描画失敗)
(b) PNG エンコード失敗		 DevTools コンソールで [s1] tile sent ログが 4 タイル × 3 パターン分出ているか確認
S4 で audioControl=false 権限ダイアログを「拒否」した iPhone 設定 → spike アプリ → マイク許可 → 再起動
Hub Portal アップロードが manifest package_id does not match the app package_id エラー 本番アプリ枠に spike ehpk を入れた spike 専用枠 (com.spwebcreat.g2runhud.spike.sN) を開いてからアップロード
