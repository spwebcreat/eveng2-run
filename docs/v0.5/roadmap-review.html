確定した制約 ─ SDK 実物で検証済

Codex 批判レビューの指摘を node_modules/@evenrealities/even_hub_sdk/dist/index.d.ts 等で全件検証済み。 これらは 動かす前提条件。設計はここから出発する。

主要制約マップ

阻害 Image container 288×144 max

SDK PB の width / height 範囲: width 20-288, height 20-144, max_count 4。

node_modules/@evenrealities/even_hub_sdk/dist/index.d.ts:393

阻害 HR API なし

SDK 全 export を grep してもハートレート関連は一切無し。内蔵 HR zone UI は嘘になるので削除必須。

grep -i "HeartRate\|heartRate\|hr" → no match

重要 Audio output なし / input あり

G2 スピーカー / 骨伝導への音声出力 API は無い。一方 AudioEventPayload（マイク PCM stream）は存在 → 音声コマンドは実現可能。

index.d.ts:860 AudioEventPayload

情報 Companion app の正式枠なし

Hub プラグインは単一 WebView エントリーポイント。「コンパニオンアプリ」とは呼ばず、その WebView をスマホ UI として活用する。

app.json: single entrypoint

重要 Locked-phone QA

Hub 審査で phone locked / backgrounded での挙動が見られる。GPS 中断・Wake Lock の補強が必須。

docs/HANDOFF.md:146 ─ 既知の弱点

重要 upng-js が toolkit deps に未記載

even-toolkit の glasses/png-utils.ts は upng-js を import するが、package.json には書かれていない。別途 install 必須。

toolkit package.json deps: react-is, recharts のみ

HR API 不在の影響

機能現状の制約復活条件分類
G2 スピーカー音声出力（TTS）SDK に音声出力 API なしSDK に audio output API が追加保留
Spotify / Apple Music 制御制御 API なし、Bluetooth 経由も実装手段なしHub plugin に音楽制御 capability 追加保留
ターンバイターン・ナビ地図 SDK なし、地図データ統合コスト過大Mapbox/Komoot 統合 or Hub 自身が map tile API 提供保留
通知転送（SMS / LINE）SDK がスマホ通知 API を持たずSDK にスマホ通知ミラーリング API が追加保留
ソーシャル機能バックエンド運用コスト過大、優先度最下位v1.0 達成後の v2 計画フェーズで再検討保留
内蔵 HR センサーSDK に HR API なしv0.9 で BLE 外部対応に確定済 → 観察対象外v0.9 BLE 代替確定
