2アップロード手順

  1. Hub Portal の 既存「G2 Run HUD」アプリ（package_id com.spwebcreat.g2runhud）に 上書きアップロードする。
  2. package_id が本番と同一なので、spike の時に出た package_id mismatch エラーは起きない（通常のバージョンアップ）。
  3. アップロード後、iPhone Hub アプリで G2 Run HUD を起動する。
新規アプリ枠ではなく 既存アプリへの上書きである点に注意。新規枠に上げると package_id mismatch が再発する。

4NG パターン（出たら即中止）

  • 起動しない / 真っ黒 / クラッシュ → version と SHA256 を確認し、g2-run-hud-0.4.0.ehpk にロールバック。
  • 心拍欄がまだ表示される → 古い build。version 0.5.0 が反映されているか Hub Portal で確認。
  • 想定外の画面 → DevTools の [g2-run-hud] ログを確認。

5DevTools 接続（補助）

  1. Mac の Safari → 開発メニュー → iPhone → 該当 WebView を選び Web Inspector を開く。
  2. console に [g2-run-hud] プレフィックスのログ（wake lock failed N times / service worker registration failed / GPX export failed 等）が出る。
注意: Even Hub アプリの WKWebView は isInspectable 未設定だと「調査可能なアプリケーションがありません」になる場合あり。その時は iPhone Hub アプリ画面の表示で代替判定する。
