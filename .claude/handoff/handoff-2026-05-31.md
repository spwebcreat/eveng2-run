## 決定事項

- **branch**: `feature/v0.5.0`（`feature/v0.4.0-design-overhaul` から作成）
- **全 6 項目は「実装済み（実機 QA 待ち）」**: コード完成 + tsc/vite build + ブラウザ standalone 確認まで済。実機 regression QA（R-1〜R-10）は未実施のため「完了」とは区別。ロードマップで緑チップ + 緑ボーダー表示
- **Service Worker は network-first**: cache-first は version bump 反映問題（古い UI 残存）と直結するため回避。登録は defensive(try/catch)。Hub WebView 上の SW 挙動は L-6 で要確認（唯一の未知数）
- **表示設定の render 伝播は optional param + デフォルト現行値**: `formatDistance(m, unit='metric')` 方式で metric 既定パスは従来と byte 一致＝regression ゼロ、imperial 選択時のみ分岐
- **旦那様指示 2 件を harness memory に保存**: 「実機確認可能になったら必ず ehpk 生成 + パス案内」（[[ehpk-on-device-ready]]）/「資料 HTML 間はナビで行き来」（[[roadmap-html-living-doc]]）

## learning 候補（今回は総量超過で追記見送り・圧縮後に追記）

1. (P1 core) Even Hub plugin に Service Worker を入れる時は **network-first** 必須（cache-first は version bump 反映問題と直結する staleness を生む）。登録は defensive、Hub WebView 上挙動は実機 QA で確認
2. (P1 core) 既存挙動のあるコードに表示オプション（単位等）を足す時は **optional param + デフォルト=現行挙動** で通す（metric 既定パスは byte 一致＝regression ゼロ、imperial 時のみ分岐）
3. (P2 workflow) 生成物の grep 検証で `grep … | head && echo OK` はパイプ末尾 head の exit 0 で **偽陽性**になる → exit code を直接評価するか `grep -c` で件数を見る
